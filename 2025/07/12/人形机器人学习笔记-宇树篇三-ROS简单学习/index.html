1. ROS基本

  1. what？ - 类似于安卓系统的一个机器人操作系统。
  2. why？ - 方便在面对不同硬件时，可以快速的搭建一个机器人（安卓手机），并可以使用社区中的node（安卓中的APP）实现不同功能（比如手机打开游戏即为游戏机，打开阅读即为书）。
  3. How？ -

1.1 ROS的基本单元：Node节点和Package包

Node：实现功能包的基本单元，如导航单元、移动单元等，但是这些节点的功能相对单一，需要很多节点相互配合才能完成一个复杂功能，因此在进行ROS相关功能下载时，不能只下载节点，而是以包的形式进行下载。

Package：多个node节点共同组合，形成具有相对复杂功能的包 - 节点的容器，ROS的基本单元。

利用catkin和cmake可以将多个node，组合成为一个包。

1.2 Publisher and Subscriber

​ ROS不同包中的节点，通过Topic和Message进行交流和信息传递。节点可以作为Publisher来创建Topic和发布消息Message；同时其他Node可以通过Subscriber订阅别的Node的Topic来进行Message接收。

1.2.1 Publisher 创建Topic并发送message

Publish Message Step：

  1. 在ROS标准Message中，选择Messgae 类型；#include <std_msgs/string.h>
  2. 初始化一个ROS的大管家；ros::NodeHandle nh;
  3. 初始化一个Pbulisher； ros::Pubsher pub = nh.advertise<std_msgs::String>(“vel”,10);//后一个参数用来设置队列长度
  4. 定义消息类型：std_mags:: String msg;
  5. 给出msg内容：msg.data = “”;
  6. 发送Message：pub.publish(mag);
  • 如果想以一定频率发送消息，则使用ros::Rate loop_rate(10);函数
  • 并在publish后，loop_rate.sleep();

1.2.2 Subscriber 订阅Topic并显示接收到的message

话题订阅步骤:

  1. 确定话题名称和消息类型

  2. 在代码文件中include<ros.h>和消息类型对应的头文件

  3. 在main函数中通过NodeHandler大管家订阅一个话题并设置消息接收回调函数。

  4. 定义一个回调函数，对接收到的消息包进行处理。

  5. main函数中需要执行ros::spinOnce()，让回调函数能够响应接收到的消息包。

rqt_graph

图形化显示当前系统活跃的节点以及节点间的话题通讯关系

快捷方法启动ros节点 - launch文件（xml文件）启动

步骤：

1.使用launch文件，可以通过roslaunch指令一次启动多个节点。
2.在launch文件中，为节点添加 output="screen”属性，可以让节点信息输出在终端中。(ROS WARN不受该属性控制)
3.在launch文件中，为节点添加launch-prefix=“gnome-terminal -e属性，可以让节点单独运行在一个独立终端中。
